下面这份清单，把“开源的网站设计相关软件”按**用途**分成 6 大类：可视化页面搭建器、静态站点生成器/前端元框架、CMS/Headless CMS、开源电商 CMS、设计/原型与图形工具、（可选）内容编辑器与代码编辑器。每项都给出**核心用途、上手难度、适合场景、亮点与注意点**，便于你快速选型。

---

# 1) 可视化页面搭建器（所见即所得 / 低代码）

适合“拖拽组件拼页面”的场景，可直接导出 HTML/CSS/JS 或嵌入到你的系统。

* **GrapesJS**（MIT）
  **用途**：Web 页面可视化搭建器，支持区块/组件体系、模板、样式与代码导出。
  **场景**：内嵌企业后台做页面搭建、邮件页/活动页搭建。
  **亮点**：纯前端、可二开扩展 Block/插件；可输出干净代码。
  **注意**：需要自行做素材管理、发布与权限体系对接。

* **VvvebJs**（MIT）
  **用途**：基于 Bootstrap 的拖拽式站点搭建。
  **场景**：快速做营销落地页、企业展示站。
  **亮点**：组件即 Bootstrap 生态，学习成本低。
  **注意**：样式多样性依赖 Bootstrap 主题。

* **Silex**（AGPLv3）
  **用途**：可视化网站构建器，支持在线编辑与静态站点发布。
  **场景**：极简个人/品牌官网。
  **亮点**：零后端、静态发布简单。
  **注意**：深度工程能力较弱，复杂交互需自己补。

> 延伸：**Editor.js**（Apache-2.0）、**CKEditor**（开源双/三许可证）、**Quill**（BSD-3）是“块状/富文本编辑器”，常嵌入你的 CMS 里做内容编辑，不是纯页面搭建器，但与页面搭建结合紧密。

# 2) 静态站点生成器 & 前端元框架（SSG/SSR/混合）

适合“内容驱动的网站（Docs/Blog/Marketing）”，或追求**极致性能**与**易部署**。

* **Hugo**（Apache-2.0，Go）
  **用途**：极快的静态站点生成。
  **场景**：文档站、博客、营销站、多语言站点。
  **亮点**：生成速度极快、主题生态丰富、部署极简（CDN 即可）。
  **注意**：复杂交互需前端自行扩展。

* **Jekyll**（MIT，Ruby） / **Eleventy**（MIT，Node） / **Hexo**（MIT，Node）
  **用途**：经典 SSG（Jekyll）、现代 SSG（11ty/Hexo）。
  **场景**：技术博客、知识库、轻量官网。
  **亮点**：主题/插件生态庞大（Hexo/11ty），易托管（GitHub Pages）。
  **注意**：对“非内容”的后台能力有限，常配 Headless CMS。

* **Docusaurus**（MIT，React） / **Astro**（MIT）
  **用途**：文档站（Docusaurus）或内容+组件混合站（Astro）。
  **场景**：企业文档门户、设计规范站、组件库官网。
  **亮点**：开箱好用、SEO 友好、可融入 React/Vue/Svelte 组件。
  **注意**：数据后台需另配（Headless CMS）。

* **Next.js**（MIT，React） / **Nuxt**（MIT，Vue） / **SvelteKit**（MIT）
  **用途**：SSR/SSG/ISR 混合的现代 Web 框架。
  **场景**：既要内容站点，又要复杂交互或登录态的官网/应用。
  **亮点**：路由/数据获取/渲染策略完备，性能与 SEO 兼顾。
  **注意**：比纯 SSG 更“工程化”，部署与运维要求更高。

# 3) CMS / Headless CMS（内容管理与多端分发）

适合“内容密集型网站”，提供后台、权限、工作流；Headless 通过 API 向前端（H5/小程序/APP）供数。

* **WordPress**（GPLv2）
  **用途**：全球最流行的 CMS。
  **场景**：官网、博客、媒体门户、营销落地页。
  **亮点**：主题/插件海量；Gutenberg 编辑器；可 Headless 化（REST/GraphQL 插件）。
  **注意**：插件治理与安全运维要到位。

* **Drupal**（GPL）、**Joomla**（GPL）
  **用途**：企业/政府网站 CMS。
  **场景**：复杂内容模型、多角色权限、工作流。
  **亮点**：可塑性/权限与结构化内容强。
  **注意**：上手曲线较陡，定制需要资深经验。

* **Ghost**（MIT）
  **用途**：极致写作与出版平台。
  **场景**：品牌内容站、订阅付费媒体。
  **亮点**：编辑体验优秀、性能好、内置会员/订阅能力。
  **注意**：偏内容出版，复杂后台逻辑需要扩展。

* **Headless：Strapi（MIT） / Directus（GPL-3.0） / KeystoneJS（MIT） / Payload CMS（MIT） / Decap CMS（MIT，原 Netlify CMS）**
  **用途**：内容建模 + API（REST/GraphQL），前后端分离。
  **场景**：需要多端分发、前端用 Next/Nuxt/SvelteKit/小程序统一消费内容。
  **亮点**：内容模型灵活、权限/工作流可定制、与现代前端天然适配。
  **注意**：需要自己搭建前台（Headless 的“自由”=更多工程工作）。

* **轻量/扁平文件：Grav（MIT）**
  **用途**：无数据库、Markdown 驱动的 CMS。
  **场景**：中小型官网、文档站。
  **亮点**：安装简单，维护成本低。
  **注意**：复杂流程与权限较弱。

* **.NET 生态：Umbraco（MIT） / Orchard Core（MIT）**
  **用途**：.NET 平台开源 CMS。
  **场景**：银行/政企的 .NET 技术栈网站。
  **亮点**：与 .NET 环境融合好、可扩展性强。
  **注意**：Windows/.NET 生态经验更友好。

# 4) 开源电商 CMS（“网站 + 商城”一体）

如果你的“网站设计”包含商品、购物车、支付等，选电商 CMS 会更省力。

* **Magento Open Source**（OSL/AFL） / **PrestaShop**（OSL） / **OpenCart**（GPL） / **WooCommerce**（GPL，WordPress 插件）
  **用途**：自建电商网站。
  **场景**：多品类商城、B2C/B2B、全球化运营。
  **亮点**：生态丰富、插件众多（支付/物流/促销）。
  **注意**：运维成本较高，主题/插件要严格筛选。

* **现代 Headless 电商：Medusa（MIT，Node） / Saleor（BSD-3，Python） / Sylius（MIT，Symfony） / Bagisto（MIT，Laravel）**
  **用途**：后端电商能力 + 前端自建（Next/Nuxt）。
  **场景**：要体验/性能/多端一致、灵活定制的电商站。
  **亮点**：API 完整、可与现代前端深度结合。
  **注意**：前端需投入更多工程量。

# 5) 设计/原型与图形工具（开源）

网页设计常用的“视觉与交互”开源工具，解决“UI 制作、图标/插画、原型交互”等。

* **Penpot**（AGPL-3）
  **用途**：开源的 Figma 替代，做 UI/原型/协作。
  **场景**：设计团队协同、设计系统建设。
  **亮点**：团队协作、组件/原型、浏览器端使用。
  **注意**：生态/插件数量少于 Figma，但在快速增长。

* **Inkscape**（GPL）、**GIMP**（GPL）、**Krita**（GPL）
  **用途**：矢量（Inkscape）、位图（GIMP）、数位板绘（Krita）。
  **场景**：图标/插画/海报/位图处理。
  **亮点**：成熟稳定、跨平台。
  **注意**：与商用软件（AI/PS）操作差异需要过渡。

* **Pencil Project**（GPL-2）、**Excalidraw**（MIT）、**diagrams.net / draw\.io**（Apache-2.0）
  **用途**：线框图/流程图/交互草图。
  **场景**：快速表达交互方案、信息架构、流程。
  **亮点**：轻量、协作友好（Excalidraw/diagrams.net 在线版）。
  **注意**：不是高保真 UI 工具，重在表达结构与流程。

# 6)（可选）编辑器与工程配套

虽然不是“设计软件”，但做网站离不开它们。

* **VSCodium / VS Code OSS**（MIT）、**Neovim**（Apache-2.0）
  前端工程事实标准，海量插件（Tailwind/ESLint/Prettier 等）。
* **Brackets**（MIT，已归档），**Atom**（归档）
  旧项目可遇到，新的不建议作为主力。

## 选型建议（如何快速定位）

* **公司需要“内容后台 + 权限 + 审核流”** → 选 **WordPress/Drupal/Ghost/Strapi/Directus**（按“是否 Headless”取舍）。
* **只做内容站/文档站/营销站**，追求**速度与易部署** → 选 **Hugo / Docusaurus / Astro**。
* **要“一套代码多端（小程序/H5）”** → 内容层用 **Headless CMS（Strapi/Keystone/Payload/Decap）**，前端用 **Next/Nuxt/SvelteKit**；小程序可引入 **Taro/UniApp（非纯开源）**。
* **要纯“拖拽搭建”** → 选 **GrapesJS / VvvebJs / Silex**，再对接你的登录、发布与素材服务。
* **需要“网站 + 商城”** → 选 **WooCommerce/PrestaShop/OpenCart**（中小体量）或 **Magento**（大体量）；想前后端分离就用 **Medusa/Saleor/Sylius/Bagisto + Next/Nuxt**。
* **设计与协作** → 选 **Penpot** + **Inkscape/GIMP/Krita** + **Excalidraw/diagrams.net**。

## 实施落地的“最小闭环”

1. **确定形态**：内容站 / 应用站 / 商城 / 混合。
2. **后端（可选）**：无后台（纯 SSG）/ 传统 CMS（WP/Drupal）/ Headless（Strapi/Directus）。
3. **前端**：Hugo/Docusaurus/Astro（内容）或 Next/Nuxt/SvelteKit（应用）。
4. **可视化搭建**：内嵌 GrapesJS/VvvebJs（如需 Drag & Drop）。
5. **CI/CD 与托管**：Git + 构建（Vite/CI）+ 静态托管（CDN）或容器化部署。
6. **监控**：接入开源监控（OpenTelemetry/自研埋点）与错误上报（Sentry 开源版）。
