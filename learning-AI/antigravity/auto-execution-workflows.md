# Antigravity 高效执行与免确认自动化指南

在 Antigravity 中，为了保障本地环境、代码和数据的安全，默认情况下对于具有潜在风险的系统操作（如执行修改环境、安装依赖、删除文件等终端命令）会要求人工确认。

如果你希望达成**最高效、无干预、免确认**的自动化任务流，最关键的方案是使用 **Workflows（工作流）** 结合 **Turbo 注解**。

以下是实现全自动执行的操作指南：

## 1. 代码编辑与常规操作（默认免确认）
对于模型的核心代码能力（如新建文件、替换部分代码片段、由于语义要求进行的代码修复等），以及只读类型的操作（如查看文件、内容检索），默认都是自动执行生效的，不需要人工点击确认。

---

## 2. 终端命令的静默执行核心：Turbo Workflows
当你需要连续执行多条需要在终端运行的命令（如构建、移动文件、部署脚本等），并且不希望每一步都弹窗等待确认时，你需要定义一个 **带有 `// turbo-all` 注解的 Workflow**。

### 如何创建免确认的工作流？
在你的项目根目录下，创建一个工作流文件，例如 `.agents/workflows/auto-build.md`（文件夹也可以叫做 `.agent`, `_agents` 或 `_agent`），内容格式如下：

```markdown
---
description: 自动构建与部署脚本
---

// turbo-all

1. 清除旧版构建：请运行 `rm -rf dist/`
2. 安装最新依赖并构建：请运行 `npm install` 和 `npm run build`
3. 复制产物并重启服务：请把生成的文件移动到目标目录，并自行重启 nginx 服务。
```

### 工作流中的两个魔法注解：
*   **`// turbo-all`**：写在文件的任意位置。只要看到这个标记，模型会在执行整个工作流期间，**强制**将所有终端命令的安全标识（`SafeToAutoRun`）置为 `true`，一切操作**完全自动执行**，不弹任何确认框。
*   **`// turbo`**：如果你只想让其中某一步（某条特定命令）自动穿透跳过确认，而其他命令依然保留人工检查，可以将 `// turbo` 写在具体步骤的正上方。

### 如何触发工作流？
一旦文件创建好，你只需要在对话中对模型说：
> *"请执行 auto-build 工作流"*

模型读取到 `// turbo-all` 后，就会像流水线一样将任务连续自动跑完。

---

## 3. 指令层面的小提示 (Prompts)
如果你没有使用 Workflow 文件，但在对话中要求运行某个**本身完全无害**的命令（例如 `echo "hello"`, `ls -la`, `git status`），模型会自行判断并设置允许自动执行。但即使你强烈要求“请直接运行且不要问我”，出于系统的硬性安全沙箱规则，只要命令存在潜在副作用（比如 `git reset --hard` 或者 `npm cache clean --force`），在不通过 Turbo 工作流的加持下模型仍会抛出弹窗等待授权。

**总结**：把流水线级操作封装到带有 `// turbo-all` 的 `.agents/workflows/*.md` 脚本中，是能够利用此 IDE 实现**真正零干预、全自动执行**的最佳方案。

---

## 4. 跳过计划模式 (Planning Mode) 的强制停止
默认规则下，遇到需要分析业务功能或修改大规模连续代码块的任务时，系统可能会触发进入 **规划模式 (Planning Mode)**。
此时模型会停手，在你的本地优先生成一份 `implementation_plan.md` 的计划书，并**强制暂停，必须等待你人为回复“同意”、“确认”后才能继续敲代码**。这可能是你感知到“分析业务代码时总需要确认”的最主要原因。

### 如何做到绝对干预豁免？
非常简单。只要你不希望模型在改代码或分析中途停下来写“实施计划”并等待你的首肯，你只需要通过提示词直接向模型下达特权指令即可。
例如，在对话开始或发送任务时附加这句话：
> **“请跳过/忽视 Planning Mode（计划模式），无需编写 implementation plan 也不要请求我的确认，想到什么就直接全速执行分析与修改。”**

只要你的需求中包含上述声明（或者你可以将它添加到你的 IDE 设置全局自定规则 User Rules 中），模型就会进入“先斩后奏”的持续执行状态，一口气帮你完成所有业务分析和代码写入，彻底消除中间的暂停确认环节。
